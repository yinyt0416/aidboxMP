小程序“语音绘打印”前端设计（uni-app版）
1. 项目概述
本项目基于 uni-app 框架开发，目标平台为微信小程序。前端主要负责：

用户账户登录（微信授权）

语音录制与上传

图片生成结果展示与管理（时间轴/画册）

图片保存到本地、分享

打印机连接管理（通过蓝牙/Wi-Fi SDK）

调用后端API完成语音识别、文生图等业务逻辑

采用 uni-app 可保证代码结构清晰，并保留未来扩展到其他小程序平台（如支付宝、百度）的可能性。

2. 页面结构设计
页面名称	路由路径	功能描述
启动/登录页	pages/login	微信授权登录，获取用户信息，跳转至首页
首页（录音）	pages/index	核心录音按钮，展示最近生成的一张图片及文字，打印机连接状态，进入历史记录入口
图片生成页	pages/generate	显示用户确认的文字，调用生成接口，展示生成中的loading及结果

图片详情页	pages/detail	展示单张图片大图及文字，提供保存、分享、删除、打印按钮
打印机管理页	pages/printer	搜索、连接/断开打印机，显示已保存的设备
个人中心页	pages/profile	用户信息、帮助与反馈、关于我们、设置等
帮助与反馈页	pages/feedback	常见问题、意见反馈表单

3. 功能模块详细设计
3.1 账户登录模块
页面：pages/login

交互：

首次进入显示微信授权按钮，调用 uni.login 获取 code，传给后端换取 openid 和 session_key。

同时调用 uni.getUserProfile 获取用户头像昵称（需用户主动点击授权）。

登录成功后跳转至首页，并缓存用户 token。

数据存储：使用 uni.setStorageSync 保存 token 和用户信息。

3.2 语音输入模块
页面：pages/index（核心录音区域）

交互：

长按录音按钮：使用 uni.getRecorderManager() 实现录音。

录音开始/结束：监听录音状态，松手后自动上传录音文件（可使用 uni.uploadFile 上传到后端 /api/speech/recognize 接口）。

上传过程中显示 loading，识别成功后返回文字，弹出确认框让用户确认或重录。

用户确认后跳转到 pages/generate，并传递识别出的文字。

优化：支持录音时长限制（如60秒），录音音量可视化。

3.3 图片生成模块
页面：pages/generate

交互：

接收上页传来的文字，显示在页面上（可编辑）。

点击“生成”按钮，调用后端 /api/image/generate 接口（POST，body 含 text）。

等待期间展示生成中动画（可用骨架屏或 loading 组件）。

成功后返回图片 URL，跳转至 pages/detail 展示结果，并将记录存入历史列表。

失败则提示错误，并提供重试按钮。

3.4 历史记录展示模块
页面：pages/history

模式切换：

时间轴模式：使用 scroll-view 纵向列表，每条记录包含缩略图、文字描述、时间，点击进入详情。

画册模式：使用网格布局（两列），仅展示缩略图，点击进入详情。

支持右上角切换视图模式（按钮切换状态）。

数据获取：首次加载调用 /api/user/history 分页获取，上拉加载更多（触底加载）。

本地缓存：可缓存最近几页数据，减少请求。

3.5 图片详情模块
页面：pages/detail

功能：

展示大图（使用 uni.previewImage 支持放大预览）。

显示对应的文字描述。

底部操作栏：保存（调用 uni.saveImageToPhotosAlbum）、分享（uni.share 或小程序菜单分享）、删除（调用 /api/image/delete，成功后返回上一页）、打印（跳转至打印机管理页，并携带图片信息）。

数据来源：通过页面参数传递图片URL和文字，或从历史记录传入ID后再请求详情。

3.6 打印机连接管理模块
页面：pages/printer

交互：

搜索设备：调用打印机 SDK 的搜索功能（如蓝牙搜索），列表展示设备名称、信号、配对状态。

点击设备：发起连接（调用 SDK 连接方法），成功后保存设备信息到本地存储，并更新首页连接状态。

断开连接：显示已连接的设备，可点击断开。

从详情页跳转过来时，可自动携带图片信息，连接成功后直接触发打印（需二次确认）。

注意：不同打印机 SDK 可能不同，需封装成统一接口。

3.7 个人中心及辅助页面
展示用户头像昵称、历史记录统计等。

帮助与反馈：常见问题列表，反馈表单提交（调用后端接口）。

关于我们：展示版本号，隐私协议等链接。

4. 与后端接口约定（简要）
以下接口需与后端协商定义，此处仅列出前端需要调用的关键接口：

接口路径	方法	请求参数	响应数据	说明
/api/user/login	POST	{ code, userInfo }	{ token, openid }	微信登录，获取 token
/api/speech/recognize	POST	file（语音文件，form-data）	{ text }	语音识别
/api/image/generate	POST	{ text }	{ imageUrl }	文生图
/api/user/history	GET	page, pageSize	{ list: [], total }	分页获取历史记录
/api/image/delete	POST	{ imageId }	{ success }	删除记录
/api/feedback	POST	{ content, contact }	{ success }	提交反馈
5. 状态管理
使用 Vuex 或 Pinia 管理全局状态：

user：token、用户信息、登录状态。

printer：当前连接的打印机设备信息、连接状态。

history（可选）：缓存的历史记录数据，避免重复请求。

loading：全局 loading 状态（如请求中）。

示例 store 结构：

js
// store/index.js
export default {
  state: {
    user: {
      token: '',
      info: {}
    },
    printer: {
      connected: false,
      device: null
    }
  },
  mutations: {
    setUser(state, user) { ... },
    setPrinter(state, printer) { ... }
  },
  actions: {
    login(code) { ... },
    connectPrinter(device) { ... }
  }
}
6. 关键实现细节
6.1 录音与上传
使用 uni.getRecorderManager() 管理录音。

录音结束后的音频文件可通过 uni.uploadFile 直接上传，同时可携带 token 进行身份认证。

注意录音格式需与后端约定（如 PCM、WAV、MP3），百度语音识别支持多种格式。

6.2 图片保存与分享
保存：使用 uni.saveImageToPhotosAlbum，需用户授权（scope.writePhotosAlbum）。

分享：通过 onShareAppMessage 定义分享内容，可设置图片、标题、路径。

对于生成的网络图片，先下载到本地临时文件再保存或分享。

6.3 打印机 SDK 集成
在 pages/printer 页面动态引入 SDK，根据打印机型号使用不同的库。

封装统一的打印机操作类（如 PrinterManager），提供 search、connect、printImage 等方法。

打印图片时，将图片URL下载为本地临时文件，转换为打印机支持的位图格式（通过 canvas 或 SDK 自带转换）。

6.4 历史记录分页
使用 onReachBottom 触底事件，调用接口获取下一页。

注意防止重复请求，使用 loadingMore 标志控制。

6.5 图片生成 loading 处理
在 pages/generate 页面进入时即显示输入的文字，点击生成后显示全屏 loading（可使用 uni.showLoading）。

若生成时间较长，可设计进度提示（如“正在绘制...大约需要5秒”）。
